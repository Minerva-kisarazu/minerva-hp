1. 前提条件・技術スタック（Technology Stack）Cline は以下のモダンなフロントエンド環境を前提として、クリーンなコードを記述すること。フレームワーク： Next.js (App Router最新安定版) または React (Vite環境)スタイリング： Tailwind CSS (ユーティリティファーストを徹底し、独自のカスタムCSSは原則禁止)言語： TypeScript (型定義を厳格に行い、any 型の使用は一律禁止)レスポンシブ設計： スマートフォン閲覧（モバイルファースト）を完全前提とし、レスポンシブブレイクポイントは Tailwind 標準（sm:, md:, lg:）を正しく使い分けること。2. ディレクトリ・コンポーネント設計（Architecture）保守性を高め、コードの重複を防ぐため、以下の構造に沿ってファイルを生成すること。共通コンポーネントの分離： 全ページ共通の「ヘッダー（Header）」および「フッター（Footer）」は、必ず components/ ディレクトリ内に独立したコンポーネントとして切り出し、各ページでインポートして使い回すこと。ページルーティング（5ページ）：/ (Homeページ)/policy (指導方針ページ)/grades (学年別のご案内ページ)/price (受講料・よくあるご質問ページ)/contact (お問い合わせ・無料体験授業申込ページ)各ページのコンポーネント、および /contact のフォームコンポーネントは、セマンティックHTML（header, main, section, footer, article, form）を正しく用いて入れ子構造を組み立てること。3. コーディング・デザイン規約（Design & Code Quality）離脱率を下げ、論理的でプロフェッショナルなブランドイメージを担保するため、以下の実装を徹底すること。余白（Spacing）の統一： セクション間の余白は十分なパディング（Tailwind の py-16 から py-24 程度）を確保し、情報が密集して読みづらくなるのを一律で防ぐこと。タイポグラフィ（Typography）の強弱： 見出し（h2, h3）と本文（p）のフォントサイズ・太さに圧倒的な強弱をつけ、スマートフォンでのスクロール時にも一瞬で重要な見出しが目に飛び込んでくる視認性を確保すること。カラーパレット： 誠実さと信頼感を表す濃紺（ネイビー）をベースカラーとし、文字はクリーンなホワイトまたは視認性の高いダークグレー、CTAボタンには視線を惹きつける目立つアクセントカラー（例：鮮やかなオレンジやゴールド系）を1色に絞って一貫して使用すること。文言の厳格な再現： 原稿ファイルに記載されている日本語テキストは、一切の省略・要約・改変をせず、そのまま完全にコード内へ実装すること。特に受講料マスタや開校時間の数値は、1文字の転記ミスも許されない。  4. コンバージョン（集客導線）とフォーム実装ルールボタンリンクの一元化： サイト内に点在するすべての「体験授業に申し込む」「お問い合わせ」「予約する」といったCTAアクションコンポーネントは、例外なく /contact ページへ遷移（Link タグを使用）させること。リンク切れやデッドリンクは絶対に発生させないこと。フォームのバリデーション（Validation）： /contact ページの入力フォームでは、必須項目（お名前、学年、電話番号、ご相談内容）が空の状態で「送信」が押された際、送信処理を中断し、各入力フィールドの直下に赤字で「お名前を入力してください」などの分かりやすいエラーメッセージを表示するフロントエンドバリデーションをJavaScript/TypeScriptで必ず実装すること。5. SEO（検索エンジン最適化）およびパフォーマンス規定メタデータの配置： 各ページのヘッド（layout.tsx または Head コンポーネント）に、地域キーワード（木更津市、金田東、個別指導塾、塾、中学生、高校受験対策など）を含んだ適切な title と description を動的に設定できるようにすること。画像の最適化： 仕様書で指定されている4枚の画像アセットエリアには、Next.js の Image コンポーネントを使用するか、imgタグに loading="lazy" を付与して遅延読み込み（Lazy Load）させ、表示速度（パフォーマンス）を極限まで高めること。
